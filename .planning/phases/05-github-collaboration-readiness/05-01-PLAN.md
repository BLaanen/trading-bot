# Plan: Portability & Configuration Cleanup

Covers ROADMAP milestones 4 (sensible defaults), 3 (.env.example), and 7 (hardcoded paths).

## Task 1: Fix hardcoded paths in shell scripts

**schedule.sh** — Line 22 has `/Users/bopeterlaanen/trading/logs`. Change to:
```
SCRIPT_DIR="$( cd "$( dirname "${BASH_SOURCE[0]}" )" && pwd )"
LOG_DIR="$SCRIPT_DIR/logs"
```
Also add `SCRIPT_DIR` resolution at the top (currently missing — the script doesn't cd to its own directory).

**monitor.sh** — Line 30 has `PY="/opt/homebrew/bin/python3.11"`. Change to match `at_open.sh` pattern:
```bash
if command -v python3.11 >/dev/null 2>&1; then
  PY="python3.11"
else
  echo "ERROR: python3.11 not found. Install with: brew install python@3.11"
  exit 1
fi
```

**eod.sh** — Line 24 has same `/opt/homebrew/bin/python3.11` hardcoded path. Apply the same fix as monitor.sh.

**Done when:**
- `grep -r '/Users/bopeterlaanen' *.sh` returns nothing
- `grep -r '/opt/homebrew' *.sh` returns nothing
- All three scripts still run correctly (test with `bash -n script.sh` for syntax check)

## Task 3: Create setup_launchd.sh for portable launchd installation

The current launchd plist files in `~/Library/LaunchAgents/` contain hardcoded paths to Bo's home directory. A new user can't just clone and run. Create `setup_launchd.sh` that generates plists dynamically.

The script should:
1. Detect `$HOME` and `$SCRIPT_DIR` automatically
2. Find the python3.11 binary path
3. Generate 3 plist files from an inline template, substituting the user's actual paths
4. Install them to `~/Library/LaunchAgents/`
5. Print instructions for `launchctl load`

Use a generic agent naming convention: `com.trading-bot.at_open`, `com.trading-bot.monitor`, `com.trading-bot.eod` instead of `com.bopeterlaanen.trading.*`.

Update `schedule.sh` AGENTS array to match the new naming.

**Plist schedule (from CLAUDE.md):**
- at_open: daily 15:25 CEST (09:25 ET) — `Hour=9, Minute=25`
- monitor: daily 18:30 CEST (12:30 ET) — `Hour=12, Minute=30`
- eod: daily 22:10 CEST (16:10 ET) — `Hour=16, Minute=10`

Note: launchd uses local time. The times above are ET. Include a comment in the generated plists explaining the timezone assumption.

**Done when:**
- `./setup_launchd.sh` generates 3 valid plists using the current user's paths
- `schedule.sh` references the new agent names
- Old personal agent names documented as migration note in setup_launchd.sh header
