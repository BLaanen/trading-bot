## Task 2: Add credential validation on startup

**File:** `orchestrator.py`

Add a new function and wire it into both `run_full_pipeline` and the `--monitor`/`--execute` flag paths:

```python
def validate_credentials(config: AgentConfig) -> bool:
    """Validate Alpaca credentials before trading. Returns True if valid."""
```

Implementation:
1. Get ALPACA_API_KEY and ALPACA_API_SECRET from env
2. If either is missing, print `[STARTUP] Missing Alpaca credentials` and return False
3. Create client via `get_alpaca_client(config)`; if None, return False
4. Call `client.get_account()`
5. Check `account.status == "ACTIVE"` and `float(account.buying_power) > 0`
6. Print `[STARTUP] Account verified: cash=${account.cash}, buying_power=${account.buying_power}`
7. If buying_power < 500, print warning but still return True
8. On any API error, print `[STARTUP] Credential check failed: {error}` and return False

Wire into orchestrator:
- In `run_full_pipeline`, add as Step -1 before regime detection (line ~316). If returns False, print abort message and return.
- In `if __name__` block, add before `--monitor` and `--execute` paths (these place orders).
- Do NOT add before `--scan`, `--report`, `--regime`, `--edge` (read-only operations).

**Done when:**
- Invalid/missing credentials cause immediate clean exit with clear message
- Valid credentials log account status
- Read-only flags still work without credentials
- No API calls are made for `--scan` or `--report`

## Task 3: Add reconciliation recovery with retry

**File:** `orchestrator.py`

Currently line 327-331 aborts the entire pipeline if reconciliation fails. Replace with retry + degraded mode:

Replace this block (lines 323-331):
```python
# Step 0.5: Broker Reconciliation
...
if not reconcile_with_broker(config):
    print("\n  ABORTING PIPELINE — reconciliation failed.")
    print("  Fix the mismatch, then re-run.")
    step_report(config, regime)
    return
```

With:
```python
reconciliation_ok = False
for attempt in range(1, 4):
    if reconcile_with_broker(config):
        reconciliation_ok = True
        break
    print(f"  [RECONCILE] Attempt {attempt}/3 failed — retrying in 10 seconds...")
    import time; time.sleep(10)

if not reconciliation_ok:
    print("\n  [RECONCILE] All 3 attempts failed — running in MONITOR-ONLY mode")
    print("  New entries blocked. Trailing stops and exits still active.")
    step_monitor(config)
    step_report(config, regime)
    return
```

This way:
- Transient API blips get 3 chances (30 seconds total)
- If still failing, existing positions are still monitored (trailing stops, bracket exits)
- New entries are blocked (no step_scan/step_execute)
- Clear log message explains what happened

**Done when:**
- Transient reconciliation failure retries 3 times
- Permanent failure enters monitor-only mode (not full abort)
- step_monitor still runs (trailing stops work)
- New entries do not execute during reconciliation failure
