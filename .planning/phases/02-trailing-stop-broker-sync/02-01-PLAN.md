# Plan: Trailing Stop Broker Sync

## Task 1: Refactor update_trailing_stops to propose (not apply) stop changes

**File:** `risk_manager.py`

Currently `update_trailing_stops` modifies `pos.stop_loss` directly (line 318). Refactor so it:
- Calculates what the new stop *would* be
- Returns the proposed new stop in `RiskDecision.new_stop` (field already exists)
- Does NOT modify `pos.stop_loss` — the caller decides after broker confirmation
- **Remove line 318** (`pos.stop_loss = round(trail_stop, 2)`) — stop mutation moves to manage_positions after broker confirmation
- Still updates `pos.high_water_mark` and `pos.trailing` (these are local-only state)
- **Add `ticker: str = ""` field to `RiskDecision` dataclass** (currently missing — needed to match decision back to position in Task 3)
- Populate `decision.ticker = pos.ticker` for every decision returned

**Done when:**
- `RiskDecision` has a `ticker` field
- update_trailing_stops returns decisions with `new_stop` and `ticker` set, but `pos.stop_loss` unchanged
- Line 318 no longer mutates `pos.stop_loss`
- Existing behavior preserved when use_trailing_stops=False (returns empty list)

## Task 2: Add _replace_stop_order to executor.py

**File:** `executor.py`

New function:

```python
def _replace_stop_order(client, stop_order_id: str, new_stop_price: float) -> tuple[bool, str]:
    """
    Replace a bracket stop-loss order at Alpaca with a new price.
    Returns (success, reason).
    
    On failure, checks order status to determine why:
    - filled: stop already triggered, exit will be handled by _check_bracket_children
    - canceled/expired: unexpected state, log warning
    - other: transient failure, retry next cycle
    """
```

Implementation:
1. Guard: if not stop_order_id or stop_order_id.startswith("SIM"): return (False, "no_order_id")
2. Try `client.replace_order(stop_order_id, stop_price=str(round(new_stop_price, 2)))` — flat kwarg, NOT nested dict
3. If succeeds → return (True, "replaced")
4. If fails → try `client.get_order(stop_order_id)` to check status
   - status == "filled" → return (False, "already_filled")
   - status in ("canceled", "expired") → return (False, f"order_{status}")
   - if get_order also fails → print `[WARN] {stop_order_id}: replace failed and status check failed: {exception}`, return (False, "status_unknown")
   - any other status → return (False, f"error: {str(e)}")

**Done when:**
- Function handles all failure paths including double-failure
- Guards against empty/SIM order IDs
- Uses correct Alpaca SDK signature (flat stop_price kwarg)
- Returns clear reason strings for logging

## Task 3: Wire trailing decisions into broker calls in manage_positions

**File:** `executor.py`

Modify Step 2 of `manage_positions`:

After calling `update_trailing_stops`, for each decision that has `new_stop` set:
1. Find the matching position by `decision.ticker`
2. If client exists (live mode):
   - Call `_replace_stop_order(client, pos.stop_order_id, decision.new_stop)`
   - If success: update `pos.stop_loss = decision.new_stop`, **call `save_positions(state)` immediately** (crash safety), print `[TRAIL] {ticker}: broker stop updated ${old} → ${new}`
   - If failed (already_filled): print `[TRAIL] {ticker}: stop already filled — exit handled next cycle`, don't update
   - If failed (other): print `[WARN] {ticker}: broker stop replace failed ({reason}) — original stop remains`, don't update
3. If no client (simulated mode):
   - Update `pos.stop_loss = decision.new_stop` directly (current behavior)

Also update `config.py`: change `trail_distance_pct` default from 0.03 to 0.05. Note: this takes effect immediately on existing trailing positions — the stop won't retroactively recalculate, but future trail moves will use the wider 5% distance.

**Done when:**
- Broker stop orders update when trailing moves the stop
- Local state saved immediately after each confirmed replace (not batched at end)
- Local state only changes on broker confirmation
- Simulated mode still works without broker calls
- Monitor log shows clear, specific messages for all outcomes
